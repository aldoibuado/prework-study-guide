HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Hypertext Markup Language (HTML) will give the content of the website structure.
  • Semantic HTML elements is the name that is given to the element that describes its content and the role it will play on the webpage.
  • The parent element holds all the children.
  • The child elements are those nested inside other elements.
  • The a element which stands for anchor will create a link to the same webpage or other webpages.
  • The HTML attribute defines additional information about an element.
  • The br element will create a line of empty space, or a line break, between two blocks of content.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much more space we want around the content inside an element.
  • The img selector will assign property values to any image element in the markup.
  • Declarations contain two important componets: the CSS property we want to apply and the value of the property.
  • Cascading Style Sheets (CSS) is a programming language used by developers to define how webpage content is presented to users.
  • Inline CSS allows you to apply style rules to specific HTML elements.
  • Internal CSS is used to define a style for a single HTML page.
  • External style sheet is a seperate CSS file that can be accessed by creating a link within the head section of the webpage.
  • Seperation of concerns refers to seperating code into different files based on use.
  • href attribute specifies the URL of the page the link goes to.
  • rel attribute specifies the relationship between the current document and the linked document resource.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • The git command must precede every Git statement.
  • Pull request or PR is used to merge a comparision branch into the base branch.
  • git pull origin main: will pull changes from the locally stored branch origin/main and merge it to the local checked-out branch.
  • git pull: will fetch and merge any commits from the tracking remote branch.
